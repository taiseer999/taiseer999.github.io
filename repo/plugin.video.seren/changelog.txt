v3.3.98
- Fix: playlist_present_check() now compares query parameters by parsed dict instead of raw string equality. Kodi normalizes URL encoding (spaces as + vs %20) between when an external addon adds the URL to the playlist and when argv[2] is passed to the plugin. The raw string compare always failed for URLs from OpenMeta (encodes spaces as %20), causing the function to fall through to create_single_item_playlist_from_info() and trigger an unnecessary re-fire of getSources via xbmc.Player().play(g.PLAYLIST). Result: every user-initiated source select from OpenMeta/OpenInfo fired getSources twice (the first invocation logged "Cancelling non playlist playback" and the second ran the actual scrape). Now both sides are parse_qsl-decoded before comparison, so encoding differences are normalized away.

v3.3.97
- Fix: Cache check grace period now correctly waits for background Tier overlap scrapers before calling _finalise_results(). Previously, when the keep-alive loop exited via timeout while Tier 1 scrapers were still running in the background overlap thread, _cache_check_done was still set (debrid workers hadn't launched yet), so _finalise_results() ran immediately with 0 cached sources on cold-cache first scrapes from Source Select / Openinfo. Grace period is now a two-stage wait: Stage 1 polls remainingProviders until all CACHE_CHECK_TIER_N sentinels drain (covers background scraping phase); Stage 2 waits on _cache_check_done (covers in-flight debrid workers). Both no-op on warm-cache runs.
